[ PHILOSOPHIE ]

BiCrypt n'est pas une application de messagerie classique. C'est un outil de communication souverain qui refuse toute dépendance aux infrastructures centralisées. Pas de serveur, pas de base de données cloud, pas de point de défaillance unique.

[ ARCHITECTURE ]

Transport exclusif via Tor V3 hidden service : chaque message transite par un circuit onion chiffré de bout en bout, sans IP exposée, sans métadonnée collectable, sans autorité intermédiaire.

POLITIQUE DE CONFIDENTIALITÉ

Dernière mise à jour : 24 avril 2026

BiCrypt est conçu selon le principe du Privacy by Design. Ce que nous ne collectons pas ne peut pas être volé, partagé ou compromis.

1. Responsable du traitement

Le traitement des données est effectué localement par l'utilisateur. Le développeur n'a aucun accès aux données créées ou échangées via l'application.

2. Données stockées sur votre terminal

  • Clés cryptographiques (Ed25519 / X25519) générées localement. La clé privée est protégée par l'Android Keystore (hardware-backed) ou l'équivalent système Desktop (libsecret / keytar).
  • Code PIN : jamais stocké en clair. Seul un hash blake2b non réversible est conservé localement.
  • Contacts (pseudonymes + clés publiques) : stockés dans la base chiffrée locale de l'application.
  • Messages : chiffrés de bout en bout (XSalsa20-Poly1305). Seule votre clé privée locale peut les déchiffrer.

3. Transit des données (le Relais)

  • Le relais (.onion) agit comme un commutateur en mémoire vive. Les messages chiffrés y transitent puis sont immédiatement supprimés — aucune persistance.
  • Aucun journal de connexion, timestamp ou identifiant n'est conservé par le relais.
  • Votre adresse IP réelle n'est jamais exposée : le relais ne voit que l'adresse d'entrée du circuit Tor (onion-to-onion).

4. Base légale

L'utilisation repose exclusivement sur votre consentement. En générant vos clés, vous initiez le traitement de vos données pour la finalité unique de communication privée.

5. Vos droits (RGPD)

BiCrypt ne détenant aucune donnée sur ses serveurs, l'exercice de vos droits s'effectue directement sur votre appareil :

  • Accès / portabilité : consultez vos messages et clés directement dans l'application.
  • Effacement : désinstallez l'application ou utilisez la suppression manuelle. La suppression est définitive — aucune sauvegarde n'existe côté développeur.

6. Sécurité

Primitives cryptographiques : X25519 (échange de clés), XSalsa20-Poly1305 (chiffrement des messages), Ed25519 (signatures numériques), blake2b-256 (hash PIN). La sécurité dépend également de l'intégrité de votre propre appareil.

7. Transferts internationaux

Le réseau Tor implique que les fragments de messages chiffrés transitent par des nœuds intermédiaires répartis dans le monde entier. Ces données sont illisibles pour les tiers et ne permettent pas de vous identifier.

8. Contact

Pour toute question relative à la protection de vos données : dépôt officiel BiCrypt sur GitHub (nefesec/bicrypt-app).
